270 /* DAN: connected to rawLLL */
271 /* Given a mutable matrix M over ZZ, and a rational number threshold, 1/4 <
272  threshold <= 1, modify M so that the columns form a Lenstra-Lenstra-Lovasz
273  basis of the image of (the original) M. ASSUMPTION: (strategy=0 case)
274  the columns of M are already a a basis for the lattice.
275  The algorithm used is that in Cohen's book on computational algebraic
276  number theory, BUT: beware of the typos in the algorithm! If there is any
277  error (interrupted, M or threshold not the correct kind), then false is
278  returned, and LLL is set to 0. If M has a column change of basis matrix
279  attached, it will be modified accordingly.
280 
281  strategy: 0 means use original Macaulay2 engine routine.
282  2 means use NTL LLL
283  (strategy%3) == 3 means use one of the real number variants:
284  GramSchmidt or Givens: 0 or 4 (respectively)
285  LLL or BKZ: 0 or 8 (respectively)
286  FP, QP1, QP, XD, RR (respectively: 0, 16, 2*16, 3*16, 4*16
287  Thus: strategy 3+4+8+16 means use NTL's GBKZ_QP1.
288 
289  For the RR variants, the suggested value of the threshold is 99/100.
290 
291  If U is not NULL, then it should be an m by m matrix over ZZ, where m is the
292  number of columns of the matrix M. In this case, it is set to be the
293  invertible transform matrix such that Mold * U = Mnew.
